## Решение 1

Для выполнения задания выполним следующие действия.

У меня уже существует зарегистрированный аккаунт на https://github.com/, поэтому сразу перейдём к созданию нового репозитория.
Для этого на сайте выберем пункт меню **"New repository"**:

![img](img/image1.png)

На открывшейся странице ведём имя репозитория **devops-netology** и выставим остальные настройки согласно заданию:

![img](img/image2.png)

Нажмём кнопку **"Create repository"**.

В следующем пункте задания нас просят создать авторизационный токен. 
В моём случае это не требуется, т.к. я уже авторизован в Github с помощью **Git Credential Manager**,
поэтому перейдём к клонированию репозитория на локальную машину.

На странице репозитория в меню **"Code"** скопируем ссылку на репозиторий:

![img](img/image3.png)

Для этого запустим консоль **Git Bash**:

![img](img/image4.png)

В консоли перейдём в директорию, в которую мы хотим склонировать репозиторий, и выполним команду:

![img](img/image5.png)

Перейдём в директорию с склонированным репозиторием:

![img](img/image6.png)

Далее в задании нас просят определить переменные user.name и user.email.
Эта настройка так же была сделана ранее и значения этих переменных можно посмотреть выполнив команду ```git config --global --list```:

![img](img/image7.png)

Выполним команду ```git status```:

![img](img/image8.png)

Отредактируем файл README.md и снова выполним команду ```git status```:

![img](img/image9.png)

В выводе мы видим, что файл README.md перешёл в состояние *Modified*, как и требуется по заданию.

Выполним команду ```git diff```:

![img](img/image10.png)

В выводе мы видим, какие изменения произошли с файлами, а конкретно в нашем случае с файлом README.md.

Выполним команду ```git diff --staged```:

![img](img/image11.png)

Команда git diff --staged показывает изменения, которые уже добавлены в staging area и будут включены в следующий коммит. 
В данный момент файл README.md находится только в состоянии Modified и ещё не был добавлен в staging с помощью git add, поэтому в staging area изменений нет и команда не выводит никаких данных.

Выполним команду ```git add README.md``` и в след за ней команду ```git status```:

![img](img/image12.png)
